How it works

A localhost daemon between agent and model.

It MITMs only your provider's API host — every other CONNECT is tunnelled through untouched.

Claude Code (or any client respecting HTTPS_PROXY) │ ▼ HTTPS_PROXY=http://127.0.0.1:8888 ┌──────────────────────────┐ │ TokenDog proxy │ localhost daemon │ • parse Messages request │ │ • filter tool_result[] │ git · gh · kubectl · terraform │ • dedup + reversible │ find · jq · curl · ~25 filters │ • re-serialize │ └────────────┬─────────────┘ │ filtered payload ▼ api.anthropic.com

Cache-safe by construction. Only the last tool_result is filtered — the part the prompt cache hasn't hashed yet. TokenDog is complementary to prompt caching, never a competitor to it.

What gets filtered

Per-tool compaction, never lossy.

Every filter holds the universal Guard invariant: output bytes ≤ input bytes. If filtering would lose data, the original passes through unchanged.

Deployment modes

Zero-config, or zero-trust. Your call.

td proxy

HTTPS_PROXY + local MITM
  • One command: td setup wires cert, auto-start & env
  • Zero client configuration
  • Best for individual developers

td gateway

NO CA CERT
SDK base_url → 127.0.0.1:8099
  • No trust-store changes, fully auditable
  • Front an existing gateway (LiteLLM, etc.)
  • Best for teams & security-conscious setups

Get started

Install in one line.

Restart your AI client, then watch the savings roll in with td gain.

$ brew tap uttej-badwane/tokendog
$ brew install tokendog
$ td setup            # cert + auto-start + proxy env, all wired

# prefer no CA cert? skip the MITM entirely:
$ td gateway --port 8099 --upstream https://api.anthropic.com
$ ANTHROPIC_BASE_URL=http://127.0.0.1:8099 claude
